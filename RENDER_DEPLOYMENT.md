## Step 1: MongoDB Atlas Setup

1. Go to **https://cloud.mongodb.com**
2. Create a cluster:
   - **Provider**: AWS
   - **Region**: Closest to your frontend
   - **Tier**: M0 Free
3. Create a Database User (remember username + password)
4. **Network Access** в†’ Add `0.0.0.0/0` (allow from anywhere)
5. Click **CONNECT** в†’ **Drivers** в†’ Copy connection string:
   ```
   mongodb+srv://USER:PASSWORD@cluster0.xxxxx.mongodb.net/
   ```
6. Replace `USER` and `PASSWORD` в†’ This is your `MONGO_URL`

## Step 2: Stripe Setup

1. Go to **https://dashboard.stripe.com**
2. Go to **Developers** в†’ **API Keys**
3. Copy:
   - `sk_test_...` в†’ `STRIPE_SECRET_KEY` (or `sk_live_...` for production)
   - Go to **Webhooks** в†’ Create endpoint for `https://your-backend.onrender.com/api/webhook/stripe`
   - Copy webhook signing secret в†’ `STRIPE_WEBHOOK_SECRET`

## Step 3: Deploy to Render

### Option A: Using Blueprint (Recommended)

1. Push repo to GitHub:
   ```bash
   git push origin main
   ```

2. Go to **https://render.com/dashboard**
3. Click **+ New** в†’ **Blueprint**
4. Connect GitHub repo
5. Render auto-reads `render.yaml` and creates 2 services

### Option B: Manual Setup

If Blueprint doesn't work:

1. Create Backend service:
   - **Type**: Web Service
   - **Runtime**: Python 3.11+
   - **Root Directory**: `backend`
   - **Build Command**:
     ```bash
     pip install --extra-index-url https://d33sy5i8bnduwe.cloudfront.net/simple/ -r requirements.txt
     ```
   - **Start Command**:
     ```bash
     uvicorn server:app --host 0.0.0.0 --port $PORT
     ```
   - **Health Check Path**: `/health`

2. Create Frontend service:
   - **Type**: Static Site
   - **Root Directory**: `frontend`
   - **Build Command**: `npm ci && npm run build`
   - **Publish Directory**: `build`

## Troubleshooting

### Backend won't start
- Check **Logs** in Render dashboard
- Common: `MONGO_URL` not set or invalid
- Check: `uvicorn server:app --host 0.0.0.0 --port $PORT` is the start command

### Health check failing
```
GET /health в†’ 500
```
Likely: MongoDB connection failed. Check:
1. `MONGO_URL` is correct
2. IP whitelist includes `0.0.0.0/0` in MongoDB Atlas

### Stripe webhook not working
1. Check webhook URL in Stripe dashboard
2. Must be: `https://career-os-backend.onrender.com/api/webhook/stripe`
3. Verify `STRIPE_WEBHOOK_SECRET` is set
4. Check backend logs for webhook errors

### Frontend can't reach backend
- Check `REACT_APP_BACKEND_URL` matches actual backend URL
- Check CORS: Backend must allow frontend origin in `CORS_ORIGINS`
- Check browser DevTools в†’ Network tab for blocked requests
